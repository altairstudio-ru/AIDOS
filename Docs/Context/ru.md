# AI Development OS — Контекст проекта (v1)

---

# Как использовать этот файл

Если ты человек:

- Это краткое объяснение всей системы
- Здесь описано, как устроен проект
- Это “карта системы”, а не код

Если ты агент:

- Используй английскую версию (en.md)
- Она является источником истины

---

# 1. Что мы строим

AI Development OS — это система, которая:

- принимает задачи на естественном языке
- автоматически понимает, что нужно сделать
- распределяет работу между агентами
- сохраняет всё в файловой системе

---

# 2. Главная идея

Вместо обычного программирования:

- человек пишет задачу
- система сама решает как её выполнить

---

# 3. Основные принципы

- Всё хранится в файлах
- Нет скрытого состояния в памяти
- Markdown — основной формат данных
- Система независима от конкретной модели (model-agnostic)
- English — технический стандарт
- Русский — для понимания человеком

---

# 4. Архитектура системы

Система работает слоями:

1. Пользователь вводит задачу
2. SCI понимает запрос
3. Workspace Loader выбирает проект
4. Task Generator создаёт задачу
5. Агент выполняет задачу
6. Результат сохраняется в файлы

---

# 5. Что такое Workspace

Workspace — это отдельный проект внутри системы.

Он содержит:

- задачи
- знания
- решения
- логи

Пример:

Workspaces/Suno-Downloader/

---

# 6. Система задач

Каждая задача — это файл:

- TASK-001.md
- TASK-002.md

Внутри:

- описание
- назначенный агент
- статус
- заметки выполнения

---

# 7. SCI (вход в систему)

SCI — это точка входа.

Он:

- принимает текст
- нормализует его
- отправляет в маршрутизатор
- запускает создание задачи

---

# 8. Агентная система

Есть разные роли:

- Executive — принимает решения
- Planner — планирует
- Research — ищет информацию
- Backend — пишет код
- Frontend — делает интерфейс
- QA — тестирует
- Reviewer — проверяет

---

# 9. Принцип работы файлов

- всё состояние хранится в файлах
- нет “магии” внутри программы
- всё можно открыть и прочитать
- система полностью прозрачна

---

# 10. Текущие решения

- Projects заменены на Workspaces
- задачи = markdown файлы
- система мультиагентная
- маршрутизация через Workspace Loader

---

# 11. Ограничения версии v1

- маршрутизация пока простая (по словам)
- нет настоящего понимания смысла
- нет event-системы
- нет памяти агентов
- нет UI панели управления

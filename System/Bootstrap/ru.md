# Механизм запуска системы

## 1. Назначение

Bootstrap Engine запускает AI Development OS из одной пользовательской цели.

Это точка входа системы.

---

## 2. Вход

Одна цель, например:

- "Сделать сервис скачивания SUNO"
- "Создать аналитическую панель"
- "Исследовать API"

---

## 3. Выход

Bootstrap Engine создаёт:

- структуру проекта
- начальный набор задач
- назначение ролей
- очередь выполнения
- первые записи в Registry

---

## 4. Процесс выполнения

### Шаг 1 — получение цели
Принимается намерение пользователя.

---

### Шаг 2 — проверка контекста
Проверяются:
- существующие проекты
- Knowledge/
- Registry
- конфликты задач

---

### Шаг 3 — запуск Task Generator
Создаётся первичный граф задач.

---

### Шаг 4 — запуск Router
Назначаются роли и создаются файлы задач.

---

### Шаг 5 — запуск Execution Loop
Система переходит в активное состояние.

---

### Шаг 6 — фиксация состояния
Обновляются:
- Progress.md
- Decisions/
- Registry

---

## 5. Ограничения

- Bootstrap не выполняет задачи
- Bootstrap не решает задачи
- Bootstrap только инициализирует систему

---

## 6. Переход состояния

INACTIVE → ACTIVE

---

## 7. Итог

Bootstrap Engine — это “кнопка включения” AI Development OS.
